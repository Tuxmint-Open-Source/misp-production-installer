# Code of Conduct

## Our pledge

We want MISP Docker Lifecycle Manager (`misp-docker-lifecycle-manager`) to be a practical, respectful, and welcoming project for people who operate, test, document, and improve MISP Docker lifecycle workflows.

We pledge to make participation in this project harassment-free for everyone, regardless of experience level, background, identity, or viewpoint.

## Expected behavior

Examples of behavior that contributes to a positive environment:

- being respectful and constructive;
- assuming good intent while still being clear about risks and defects;
- giving actionable feedback;
- accepting maintainer decisions after discussion;
- protecting sensitive operational details when reporting issues.

## Unacceptable behavior

Examples of unacceptable behavior:

- harassment, insults, threats, or personal attacks;
- publishing another person's private information;
- posting credentials, tokens, private keys, private hostnames/IPs, internal topology, or raw logs that may expose someone else's deployment;
- repeatedly derailing technical discussions;
- other conduct that maintainers reasonably consider inappropriate.

## Enforcement

Maintainers may remove, edit, or reject comments, commits, issues, pull requests, or other contributions that violate this Code of Conduct.

Maintainers may also temporarily or permanently ban contributors for behavior that is harmful, unsafe, or inappropriate.

## Reporting

If you see behavior that violates this Code of Conduct, report it to the project maintainers through GitHub's reporting tools or by contacting the repository owner through their public GitHub profile.

For security vulnerabilities, do **not** use the Code of Conduct process. Use the security reporting path described in the repository security policy instead.

## Scope

This Code of Conduct applies to project spaces, including issues, pull requests, discussions, release comments, and other public project interactions.
